# AGENTS.md

## 前端界面约束
- 业务软件主界面只展示业务对象、业务状态和用户可以直接完成的业务动作。
- 禁止在业务主界面展示“系统状态”“服务状态”“刷新数据”“后端地址”“接口地址”“端口”“模型名称”“Jenkins”“Gitea”等开发、运维或基础设施信息。
- 需要排查或运维时，相关信息只能放在 README、日志、明确的开发/运维调试页或命令行输出中。
- 遇到用户质疑界面晦涩时，先判断该 UI 是否服务真实业务动作；不服务业务动作的控件应删除，而不是改名保留。

## 交付与打包约束
- 重新生成桌面端、安装包或其他可交付产物时，如果旧产物目录被本项目进程占用，应先定位占用 `release`、`dist` 或目标产物目录的本项目进程，并主动关闭这些旧进程后继续打包。
- 关闭进程前必须确认进程路径属于当前项目产物目录，禁止误杀无关程序。
- 生成新产物后只保留当前最新可交付版本，避免用户误用旧产物。

## 版本提交约束
- 代码提交信息必须使用中文，标题和正文都应直接说明本次有效变更。
- 禁止使用英文前缀或英文提交标题。
